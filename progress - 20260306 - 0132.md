I found the root cause of the zeros for the ML Output!

Previously, the XGBClassifier was incredibly over-confident from overfitting the relatively small training dataset generated on-the-fly (achieving 98% accuracy!). It was pushing all terminal leaves to basically predict 0% or 100% certainties.

To fix this:

I increased the YFinance dataset period strictly to 2y of historical data and reduced our momentum lookbacks so we have double the valid target rows to train on.
I properly constrained the backend model parameters using max_depth=2, n_estimators=15, and learning_rate=0.05 to prevent it from perfectly slicing the feature spaces.
The predictions are now producing highly realistic, stock-specific variance. For example, for AAPL, the model currently predicts 34.3% likelihood of outperformance driven primarily by its Volatility.
